Screen 1Profile Management
profile_management
Purpose
Identity and area-assignment review (Section 11.1).
Business Requirement
  • Displays Scout ID and assigned districts (read-only).
  • Allows editing of name and contact details.
User Experience (UX)
Circular avatar. Detail rows with edit icons. "Assigned Areas" map-link or list.
User Actions
  • Update Name.
  • View Assignments.
Technical Details
Reads from `scouts` table.
Database Schema
SELECT * FROM scouts WHERE id = $1;

Screen 2App Settings
app_settings
Purpose
Localization and system preferences (Section 11.3 - 11.5).
Business Requirement
  • Language: English, Hindi, Bengali.
  • Offline Mode: Toggle for data-saving behavior.
  • Notifications: Push alerts for new tasks.
User Experience (UX)
Switch-based settings list. Language picker modal. Storage usage indicator.
User Actions
  • Change Language.
  • Toggle Notifications.
  • Clear Local Cache (Safe wipe).
Technical Details
Stored in `SharedPreferences` / `LocalStorage`. Triggers immediate i18n update.

Screen 3Auth & Logout
auth_logout
Purpose
Session termination (Section 11.7).
Business Requirement
Logout must clear sensitive local task data. Cannot logout if sync queue is not empty (Warning).
User Experience (UX)
Danger-red "Logout" button. Confirmation dialog. "Caution: Pending Sync" warning if applicable.
User Actions
  • Confirm Logout.
Technical Details
Clears JWT and secure storage. Navigates back to Splash.
