Backend Architecture & Implementation Guide

For Developers — Complete Specifications for Backend Implementation

tRPC Procedures (5 Screens)

Re-inspection mirrors first inspections but maintains relational bindings to target preceding failure logs.

Lineage Strategy:
• Insert-only policy. Commits a completely new record in inspections setting prior_inspection_id and incrementing reinspection_seq.
• Allows deep walking histories while preserving initial evidence blocks.

Key Procedures

  • scout.lot.initReinspection — Protected. Fetches historical inspections lists, grabs preceding parameters, and loads locked specifications version.
  • scout.lot.submitReinspection — Protected. Atomic multi-table write. Sets reinspection_seq = prior.reinspection_seq + 1. Writes new inspections row, child readings, and defects.
  • scout.media.getUploadUrl — Protected. Issues a short-lived Cloudflare R2 presigned POST URL for direct photo upload from the mobile device. Returns { uploadUrl, fileKey }.
  • scout.media.confirmUpload — Protected. Registers the uploaded file's R2 key and content hash into the relevant inspection photo record after client confirms successful upload.

Database Schemas (Drizzle ORM DDL)

inspections (Relational Lineage Columns Added)

ALTER TABLE inspections ADD COLUMN prior_inspection_id UUID REFERENCES inspections(id), ADD COLUMN reinspection_seq INT NOT NULL DEFAULT 0; -- Lineage walking index CREATE INDEX idx_insp_lineage ON inspections (prior_inspection_id, reinspection_seq);

inspection_parameter_readings (Quality Snapshot)

-- Written post-reconditioning to store new metrics: CREATE TABLE inspection_parameter_readings ( id UUID PRIMARY KEY DEFAULT gen_random_uuid(), inspection_id UUID NOT NULL REFERENCES inspections(id) ON DELETE CASCADE, parameter_code TEXT NOT NULL, value_numeric NUMERIC(7,2), band TEXT CHECK (band IN ('A','B','C','out')), spec_version INT NOT NULL );

Row-Level Security (RLS)

-- Re-inspections inherit the same scope guard as initial inspections CREATE POLICY scout_reinspection_scope ON inspections USING ( warehouse_id IN ( SELECT warehouse_id FROM scout_assignments_warehouses WHERE scout_id = current_user_id() AND active = true ) ); -- Prior failing inspection is immutable — no UPDATE permitted for non-admin roles REVOKE UPDATE ON inspections FROM scout_role;

Strict Input Validations & Error Enforcements

  • Failed Metrics Check: Enforces that previously failing parameters must be re-entered. If moisture was the root rejection reason, moisture measurement inputs are forced.
  • Sequence Warning: If reinspection_seq ≥ 2, server raises administrative warnings indicating chronic lot failures.

Offline Caching & Queue Synchronization

  • Offline State: Saves logs locally, holding temporary sequence tags.
  • Notification Triggers: Syncing accepted re-inspections closes the rejection notice, updates lot state, and generates updated certificates carrying reinspection_seq meta tags.

Security & Media Protection Protocols

  • RLS Boundaries: Only assigned quality inspectors have credentials to declare reinspection verdicts.
  • Immutability: The prior failing inspection record is set read-only. No updates can mutate historical results.
