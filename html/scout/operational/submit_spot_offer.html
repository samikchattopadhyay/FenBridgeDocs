Backend Architecture & Implementation Guide

For Developers — Complete Specifications for Backend Implementation

tRPC Procedures (6 Screens)

This proxy wizard facilitates registering spot supply lots for offline farmers, triggering quality visits.

State Operations:
• Submission inserts records in spot_listings, and if pond-sourced, generates corresponding lots records linked as pending_verification.

Key Procedures

  • initProxySpotOffer — Protected. Fetches assigned farmers list and queries corresponding active pond indices.
  • getScoutAvailability — Protected. Computes available inspection calendar slots based on district calendars and travel distances.
  • submitProxySpotOffer — Protected. Atomic mutator. Inserts a new row in spot_listings, initializes a child lots row, and logs a scheduled visit_schedules event.

Database Schemas (Drizzle ORM DDL)

spot_listings (Market Registry Header)

CREATE TABLE spot_listings ( id UUID PRIMARY KEY DEFAULT gen_random_uuid(), farmer_id UUID NOT NULL REFERENCES farmers(id), lot_id UUID NOT NULL REFERENCES lots(id), scout_id UUID REFERENCES scouts(id), status TEXT NOT NULL DEFAULT 'pending_verification' CHECK (status IN ('pending_verification','listed','reserved','sold','cancelled')), scout_visit_scheduled_at TIMESTAMPTZ NOT NULL, estimated_parameters JSONB NOT NULL, created_at TIMESTAMPTZ NOT NULL DEFAULT now() ); CREATE INDEX idx_spot_listings_farmer ON spot_listings (farmer_id); CREATE INDEX idx_spot_listings_status ON spot_listings (status);

visit_schedules (Audit Verification Booking)

CREATE TABLE visit_schedules ( id UUID PRIMARY KEY DEFAULT gen_random_uuid(), farmer_id UUID NOT NULL REFERENCES farmers(id), scout_id UUID NOT NULL REFERENCES scouts(id), pond_id UUID REFERENCES ponds(id), scheduled_for TIMESTAMPTZ NOT NULL, purpose TEXT NOT NULL CHECK (purpose IN ('pond_inspection','lot_verification','seal_application')), status TEXT NOT NULL DEFAULT 'scheduled' CHECK (status IN ('scheduled','in_progress','completed','cancelled')), spot_listing_id UUID REFERENCES spot_listings(id) );

Strict Input Validations & Error Enforcements

  • Volume Bounds: Estimated volumes must be greater than zero. Volume declarations above typical capacities of the chosen pond raise warnings.

Offline Caching & Queue Synchronization

  • Draft Caching: Details cache locally inside SQLite. Sync requests write scheduled inspection slots on server databases if slots are still open.

Security & Media Protection Protocols

  • Zone Enforcement: Only permits submitting offers for farmers registered inside the scout's designated zone bounds.
Submit Spot Offer Steps Directory
6 Steps
1 Select Farmer User
Purpose
Identifies the specific offline farmer for whom the scout is submitting the spot listing offer.
Business Requirement
Scouts are restricted to selecting active, registered farmers within their designated operational zone.
User Experience (UX)

Searchable index with real-time suggestions.

Details cards display active ponds, coordinates, and pending validation flags to help scouts select the correct account.

Technical Details
Validates and saves the target farmer_id in the current wizard memory block.
2 Select Supply Source
Purpose
Identifies the source pond or temporary storage warehouse supplying the crop.
Business Requirement
Guarantees crop tracking by requiring scouts to tie the spot listing directly to a registered asset.
User Experience (UX)

Grid layout lists all active ponds and warehouses registered under the selected farmer.

Cards detail capacity, coordinate, and landmark data.

Technical Details
Stores the selected pond_id or warehouse_id variables in local memory.
3 Lot Details & Grade Expectations
Purpose
Declares estimated crop volumes and tentative quality grade parameters.
Business Requirement

Establishes initial quantity expectations.

Grade estimations can be marked "Unsure" to await official testing.

User Experience (UX)

Input panels for total kg and moisture estimations.

Segmented switches pick expected grades (A/B/C/Unsure).

Bottom CTA advances.

Technical Details

Validates volume limits against standard capacities.

Saves parameters in wizard states.

4 Crop Quality Photos
Purpose
Captures raw crop seed photos to provide visual validation to prospective digital buyers.
Business Requirement
Allows attaching supporting documents or photos showing the raw crop quality to increase selection chances.
User Experience (UX)

Camera layouts trigger camera feeds.

Image grids preview raw seeds quality.

Supports quick deletions and re-capturing.

Technical Details
Caches image binary arrays in local database tables until sync execution.
5 Schedule Quality Verification Visit
Purpose
Schedules mandatory verification visits by Quality Inspectors to inspect and seal the lot.
Business Requirement

Validates inspection slots against district calendar models.

Verification checks are mandatory before market publication.

User Experience (UX)

Horizontal scrolling selector maps dates.

Hour chips highlight available inspector slots.

Selection actions pop verification summaries.

Technical Details

Queries getScoutAvailability routines.

Caches schedule coordinates.

6 Review & Submit Spot Offer
Purpose
Summarizes spot metrics and commits the proxy spot listing contract.
Business Requirement

Atomic persistence.

Sets state to pending_verification, prompting push notices to scheduled inspectors.

User Experience (UX)

Summary card displays all declared metrics and scheduled visit details.

Signature confirmation checkbox.

Cyan CTA commits spot request.

Centered confirmation panel shows progress.

Technical Details

Invokes submitProxySpotOffer to execute database transactions.

Resets wizard parameters.
