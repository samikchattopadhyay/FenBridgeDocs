Screen 1My Spot Listings Repository
my_spot_listings
Purpose
Show all spot-channel listings the farmer has requested or that are currently active. Track each lot from request → scout verification → seal applied → listed → reserved → sold.
Business Requirement
Spot stock is BM's second sales channel. Farmer prepares a ready lot, scout verifies and applies a tamper-evident seal. Listing premium is applied. Paid when lot sells. Seal breaking voids the listing.
UX & User Actions
UX: Filterable list of spot listings. Each card shows lot state prominently.

Actions:
• Filter by status: All · Pending Verification · Listed · Reserved · Sold · Expired · Rejected.
• Search by listing ID or lot reference string.
• Tap row to open Spot Listing Detail sheet.
• Trigger primary Request Spot Listing flows via FAB.
Technical & Design Details
• Query: SELECT * FROM spot_listings WHERE farmer_id = current_user.
• Listing price displayed is BM-to-farmer amount, fully protecting buyer wholesale margins.
• Real-time updates track shelf-life expiration timers.
Database Schema
lots (id UUID PK, farmer_id UUID, quantity_kg INT, origin ENUM); spot_listings (id UUID PK, lot_id UUID, status ENUM, shelf_life_days INT, farmer_payable NUMERIC); spot_listing_quality_report (id UUID PK, spot_listing_id UUID, moisture_pct DECIMAL);

Screen 2Spot Listing Detail Sheet
spot_listing_detail
Purpose
Full read-out of a single spot listing with state-dependent content — scout visit schedule, seal evidence + verification report, reservation/sale links, or rejection reason.
Business Requirement
Physical verification + tamper-evident seal builds buyer trust. Producers need clear visual access to seal imagery, laboratory grading metrics, and absolute payout triggers. Preserves absolute margin masking.
UX & User Actions
UX: Single scrollable screen with dynamically conditional status views.

Actions:
• View full-screen tamper-evident seal gallery photos.
• Inspect detailed lab moisture and size verification matrices.
• Cancel pending requests or re-request with corrections.
• Track live reservation windows.
Technical Details
Loads single row via SELECT * FROM spot_listings joined with lots and quality reports. Serves signed secure URLs for image viewports with 1-hour expiry TTLs.
Row Level Security Mapping
-- Strips market listing pricing cleanly before client transport: SELECT id, lot_id, status, seal_id, shelf_life_days, farmer_payable, expires_at FROM spot_listings WHERE id = ? AND farmer_id = current_user;
