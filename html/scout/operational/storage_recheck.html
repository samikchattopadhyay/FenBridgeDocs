Trigger Conditions & Light Check Pattern

When Triggered
Lot crosses ~50% shelf-life unsold. Admin pushes re-check task to scout. Alternatively, triggered by anomaly report from warehouse.
Light Check Pattern
NOT a full re-grade. Focuses on seal integrity, net condition, and basic storage state. Only escalates to full re-inspection if anomaly found.
Offline-First
All data captured locally; rides existing scout sync_queue. Designed for offline-capable inventory management.
Immutability
Once outcome recorded, scout cannot edit. Admin can correct via backend only.
