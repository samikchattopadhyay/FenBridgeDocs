For Developers — Complete Specifications for Backend Implementation

tRPC Procedures (5 Screens)

This proxy wizard facilitates marking production complete for offline farmers, updating logistics pipelines.

State Precondition:
• Purchase Order state must be exactly in_production to permit fulfillment readiness logs.

Key Procedures

  • scout.lot.initProxyFulfillment — Protected. Fetches active purchase_orders locked in production state for target farmers.
  • scout.lot.markReady — Protected. Atomic mutator. Set PO state to ready, populates bagging metrics, logs photos, and inserts lot_ready_events rows.
  • scout.media.getUploadUrl — Protected. Issues a short-lived Cloudflare R2 presigned POST URL for direct photo upload from the mobile device. Returns { uploadUrl, fileKey }.
  • scout.media.confirmUpload — Protected. Registers the uploaded R2 file key and content hash into lot_photos after client confirms successful upload.

Database Schemas (Drizzle ORM DDL)

lot_ready_events (Audit Trail Ledger)

CREATE TABLE lot_ready_events ( id UUID PRIMARY KEY DEFAULT gen_random_uuid(), po_id UUID NOT NULL REFERENCES purchase_orders(id), farmer_id UUID NOT NULL REFERENCES users(id), bagged_quantity_kg INT NOT NULL, bag_count INT NOT NULL, marked_ready_at TIMESTAMPTZ NOT NULL, marked_ready_by_user_id UUID NOT NULL REFERENCES users(id), created_at TIMESTAMPTZ NOT NULL DEFAULT now() ); CREATE INDEX idx_lre_po ON lot_ready_events (po_id);

lot_photos (Readiness Image Catalog)

CREATE TABLE lot_photos ( id UUID PRIMARY KEY DEFAULT gen_random_uuid(), po_id UUID NOT NULL REFERENCES purchase_orders(id), farmer_id UUID NOT NULL REFERENCES farmers(id), storage_path TEXT NOT NULL, uploaded_at TIMESTAMPTZ NOT NULL, uploaded_by_user_id UUID NOT NULL REFERENCES users(id) );

Row-Level Security (RLS)

-- Proxy fulfillment: scout must be assigned to the lot's warehouse CREATE POLICY scout_lot_ready_write ON lot_ready_events FOR INSERT WITH CHECK ( declared_by_scout_id = current_user_id() AND lot_id IN ( SELECT l.id FROM lots l JOIN scout_assignments_warehouses sa ON l.warehouse_id = sa.warehouse_id WHERE sa.scout_id = current_user_id() AND sa.active = true ) );

Strict Input Validations & Error Enforcements

  • Weight Shortfall Flags: bagged_quantity_kg must be less than or equal to PO specifications. shortfalls trigger warnings; excess weight declarations are locked.
  • Bag Count Bounds: Minimum 1 bag. Asserts that average bag weight is within 30kg - 60kg limits.

Offline Caching & Queue Synchronization

  • Offline Buffer: Compiles photo files to local directory queues, executing server-side uploads and schedule synchronization once stable internet returns.

Security & Media Protection Protocols

  • Proxy Authorization check: Asserts that only assigned scouts can invoke transactions on behalf of farmers inside their districts.

1 Select Purchase Order
Purpose
Identifies which active Purchase Order this ready consignment is fulfilling.
Business Requirement

Prevents accidental logs.

Limits options exclusively to POs assigned to the scout's district currently in production.

User Experience (UX)

Searchable card list of POs, sorted by target ready dates.

Cards show farmer name, target weights, and grades.

Selection enables the bottom sticky "Continue" CTA.

Technical Details
Loads PO parameters into local wizard context buffers.
2 Quantity Bagged
Purpose
Registers the total volume and physical bag counts packed and waiting at the farm.
Business Requirement

Logs total bagged quantity in kilograms and specific bag counts.

bag weight variance rules are applied client-side.

User Experience (UX)

Top card recaps PO expectations.

Numeric inputs for total kg and bag count are provided.

Displays live calculated average per-bag weights.

Technical Details
Restricts submissions if input exceeds PO volume specifications.
3 Confirm Bag Count
Purpose
Ensures a highly detailed audit log of individual bags.
Business Requirement

Forces validation checks showing that no single bag exceeds tolerance thresholds.

Ensures consistent scaling practices.

User Experience (UX)

Renders an interactive list matching the declared bag count, allowing quick adjustments to individual bag weights.

Dynamic indicators flag uneven weight distributions.

Technical Details
Calculates standard deviations to verify packaging uniformity before advancing.

Captures photographic proof of the packed inventory to justify scheduling dispatch logistics.
Business Requirement

User Experience (UX)

Action triggers opening device camera and photo files picker.

Thumbnail lists show uploaded files.

Primary button unlocks when criteria is resolved.

Technical Details

Validates file sizes.

Saves paths to device memory context.

5 Notify Scout & Confirm
Purpose
Finalizes the proxy declaration, updates ledgers, and triggers verification scheduling.

Automatically inserts schedule parameters to trigger verification visits within 24 hours.
