Core Authentication

Entry & Auth (Mobile + OTP)

Universal authentication flow supporting all user roles (scouts, farmers, poppers, buyers) via mobile number and SMS-based OTP verification with role-based dashboard routing.

Workflow Overview

Mobile + OTP is the sole authentication path for FenBridge. Country defaults to India (+91); 10-digit Indian mobile validated. The same entry screen serves all roles — after successful authentication, users are routed to their role-specific dashboard. For newly registered users (farmers/poppers), scouts register them on their behalf, and they receive their profile pre-populated when they first log in. Consent is captured explicitly, and OTP verification is real-time SMS delivery.

Auto-detect existing session and skip login when a valid token exists; otherwise route to mobile entry. Maximum ~1.5s.

2Mobile Entry
Purpose
Capture the user's mobile number and request an OTP.
Business Requirement
Mobile + OTP is the sole auth path. Country defaults to India (+91); 10-digit Indian mobile validated. Rate-limited to prevent OTP spam. The same screen serves all roles — Splash routes to dashboard by role after auth succeeds.
User Experience (UX)
Single-field screen. Auto-focus, numeric keypad. Continue enabled on valid 10-digit number. Helper line clarifying "We'll send you a 6-digit OTP".
User Actions
• Enter 10-digit mobile
• Tap "Send OTP" → OTP Verification screen
• Tap country code → country picker (rarely used in Year 1)
Technical Details
Validation: Indian mobile regex ^[6-9][0-9]{9}$. Calls Supabase auth.signInWithOtp({ phone }) with phone in E.164 (+91XXXXXXXXXX). Supabase forwards OTP via configured SMS provider (MSG91 / Twilio). Rate limit: 3 OTPs per number per hour (server enforced); client shows a friendly 60s cooldown after each send. No user record needs to pre-exist — Supabase creates an auth row on first send; the FenBridge profile (users table) is created/linked on first successful verify. Fully offline-incompatible (requires network); offline message: "You need to be online to log in."
Design Details
Top bar: none (this is the entry screen). Brand mark (small, top-centre). Headline: "Log in to FenBridge" (Title 2). Sub-line: "Enter your mobile number — we'll send you an OTP" (text-md). Input: country chip (+91) + 10-digit number field. Primary CTA: "Send OTP" — full-width, disabled until valid. Footer line: "New users? Speak to a Bharat Makhana scout to get started." (text-md).
Database Schema
No new tables. Uses Supabase auth.users (managed) and the existing users table that links auth ID to FenBridge profile. The users table must expose: id, auth_user_id, role, mobile, mobile_verified_at, status (pending_approval / active / suspended), created_via_scout_id (nullable, set for scout-assisted onboarding).
3OTP Verification
Purpose
Verify the 6-digit OTP sent to the entered mobile.
Business Requirement
Successful verify creates/links the FenBridge users row, resolves the role, and routes onward. Failed OTP shows a clear error without leaking whether the number is registered. Resend timer prevents abuse.
User Experience (UX)
Six-box OTP input, auto-advance, paste-detection. Resend visible after 30s. Mobile number echoed at top with an edit shortcut.
User Actions
• Enter OTP (auto-advance per digit)
• (Auto-verify on 6th digit, or tap "Verify")
• On success → role-routed dashboard
• On fail → clear field + retry; after 3 failures, brief lockout message
• "Resend OTP" after 30s
• "Change number" → back to Mobile Entry
Technical Details
Calls Supabase auth.verifyOtp({ phone, token, type: 'sms' }). On success: receives Supabase JWT; lookup/create users row keyed on auth_user_id. Role resolution: read users.role; if missing (brand-new auth user, no profile), default to a "pre-role" state that surfaces a "Contact a scout to set up your account" message — because, per the new model, regular users (farmers/poppers/buyers) are registered by scouts, not self-served. Successful scout login: route to Scout Dashboard. Token persisted in local secure storage; refresh handled by Supabase SDK. Auto-read OTP from SMS where Android Autofill API supports it.
Design Details
Top bar: back arrow, no title. Headline: "Enter the OTP" (Title 2). Sub-line: "Sent to +91 98765 43210" + "Change" text button. Six-box OTP input (large, monospace). Inline error (danger): "Incorrect OTP — try again". Resend row: "Resend OTP in 0:24" (disabled) → "Resend OTP" (enabled after 30s). Primary CTA: "Verify" — auto-fires on 6 digits filled. Footer line: "Make sure your network can receive SMS." (text-md).
Database Schema
Reads/writes users (links auth_user_id, sets mobile_verified_at). No new tables.
