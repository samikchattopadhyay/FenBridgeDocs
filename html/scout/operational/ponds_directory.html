Workflow Module 3

Ponds Directory

Browse all assigned ponds with inspection history. Lists ponds scout is assigned to with search and filter capabilities. Detailed pond profiles showing inspection history, spot offers, and quick access to start new inspections.

Directory Screens
3 Screens
Screen 1Pond List
Purpose
Browse all assigned ponds with search and filter capabilities.
Features
  • Search bar (by farmer name, pond code, village)
  • Filter chips by inspection recency (Today, This Week, Overdue)
  • Card layout showing: farmer name, pond code, village, area, last inspection date
Scope
Scout-assigned ponds only; populated from scout_assignments_ponds table.
User Actions
Search · Filter · Tap card → Pond Detail
Screen 2Pond Detail
Layout
Header (pond name, farmer, area) + Tabbed content (Profile · Inspections · Spot Offers)
Tab: Profile
  • Pond name, farmer contact
  • Area, unit (Acre/Bigha/Hectare)
  • GPS location with map
  • Address and landmark
  • Legal document proof indicator
Tab: Inspections
Chronological history of all pond inspections (newest first). Show date, scout name, water level, algae, readiness score, photo count.
Tab: Spot Offers
All spot_offers linked to this pond source. Show lot details, pricing, status (pending/verified/sealed/sold), and listing date.
Screen 3Pond Inspection Record (Read-Only)
Purpose
Full view of a single past pond inspection; audit trail for learning data.
Content Shown
  • Inspection metadata (date, scout, arrival GPS)
  • Conditions summary (water level, algae, pest flags, readiness)
  • Bund readings table with average displayed
  • Photo strip (all 1-3 photos from inspection)
  • Notes preview or full text
Requirement
Read-only; no edits allowed. Scout cannot modify past inspections after submission.
Offline
Cached locally; photos available offline if previously synced.

Key Design Patterns

Scope Limitation
Scout sees only ponds assigned to them. No cross-territory visibility. Enforced server-side.
Tabbed Navigation
Profile, Inspections, and Spot Offers organized as tabs for clean information hierarchy.
Learning-Only Purpose
Pond inspections capture supply intelligence (Year 1 learning). Not yet transactional or commercialized.
Inspection Immutability
Once submitted, inspection record is read-only. Scout cannot edit; Admin-only corrections.
