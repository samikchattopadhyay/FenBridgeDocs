TECHNICAL SPECIFICATIONS
================================================================================

UX/UI Design
  • Dark mode with Scout accent (#6ffbbe)
  • Hanken Grotesk headers, Inter body text
  • Static headers + flowing footers
  • Stepper progress bar (X of 7)
  • Consistent component library

Offline Architecture
  • Local cache: Hive (UI state) + SQLite (sync queue)
  • Background sync: 30s interval
  • Conflict resolution: last-write-wins + audit trail
  • No data loss on network interruption

GPS Integration
  • Haversine distance calculation
  • Accuracy validation (target: ≤50m)
  • Warning thresholds & user feedback
  • EXIF metadata from photos

Grade Calculation
  • Multi-parameter logic: defects %, moisture, weight, hardness
  • Server-side RPC for consistency
  • Real-time UI feedback
  • Grade A/B/C/Reject classification

Photo Management
  • Local filesystem storage
  • SQLite index for sync queue
  • Compression to 2MB target
  • Cloud upload on reconnection
  • EXIF preservation

Database Schema
  • 6 core tables: lot_inspections, lot_inspection_defects,
    lot_inspection_photos, lot_inspection_reference_samples,
    scout_tasks, admin_approval_queue
  • Foreign key relationships
  • Check constraints for enum validation
  • Timestamps for audit trail
  • Indexes for performance (scout_id, status, due_date)
