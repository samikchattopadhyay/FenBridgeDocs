Screen 1Aggregated Earnings Summary
earnings_summary
Purpose
Aggregated earnings view across all channels and time periods. Help the farmer understand performance, compare months, and make planning decisions.
Business Requirement
Transparent visibility into cumulative financial outcomes incentivizes consistent volume throughput. Adheres strictly to learning-only metrics without scoring manipulation.
UX & User Actions
UX: High-level statistical cards coupled with simple graphical readouts.

Actions:
• Toggle view interval: This Month / Last 3 Months / This Year / All Time / Custom.
• Tap individual channel or grade summary segments to open pre-filtered Payment History matrices.
Technical Details
Queries materialized reporting views or triggers server-side aggregation via Supabase RPC farmer_earnings_summary(user_id, from_date, to_date).
Database Schema
-- Materialized view utilized for rapid timeline analytical aggregations: farmer_earnings_mv (farmer_id UUID, period_month DATE, channel ENUM, grade ENUM, total_received NUMERIC);

Screen 2Chronological Payment History
payment_history
Purpose
Chronological record of all payments BM has made to this farmer, with full traceability (UTR, date, amount, linked PO/spot listing).
Business Requirement
Guarantee absolute auditing transparency for downstream fund reconciliations. Disputes resolved entirely by persistent UTR logging.
UX & User Actions
UX: Reverse-chronological ledger categorized by distinct processing months.

Actions:
• Search ledger entries by absolute UTR string.
• Filter records by channel leg origin (PO / Spot / Advance).
• Load granular clearing proof details.
Technical Details
Queries confirmed payments where payer party is bm_to_farmer joined with linked order references. Exposes ledger status purely as an authoritative auditing record.
Database Schema
-- Fetch sequence retrieving cleared counterparty disbursements: SELECT * FROM payments WHERE payer_party = 'bm_to_farmer' AND farmer_id = current_user AND status = 'confirmed' ORDER BY confirmed_at DESC;

Screen 3Payment Receipt Detail
payment_receipt_detail
Purpose
Printable and shareable single source of truth detailing clearing dates, encrypted receiving bank numbers, and absolute UTR proof strings. Fully matches standard bank statement feeds.
Business Requirement
Serves as a standalone bank-level transfer ledger receipt. Facilitates off-platform stakeholder reference and auditing confirmation.
UX & User Actions
UX: High-contrast receipt card highlighting cleared transfer variables.

Actions:
• Copy full length bank clearing UTR string directly to native device clipboards.
• Trigger standard OS share menu actions for receipt export.
Technical Details
Fetches target row details from payments enforcing strictly producer access scope boundaries.
Database Schema
payments (id UUID PK, po_id UUID, spot_listing_id UUID, expected_amount NUMERIC, utr TEXT, status ENUM, confirmed_at TIMESTAMP);
