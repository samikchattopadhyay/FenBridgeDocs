Screen 1Confirm Lot & Quantity
confirm_lot_quantity
Purpose
Validation of the physical lot against the digital declaration. Confirms total bag count and net weight before quality measurement.
Business Requirement
  • Locked weight becomes the transaction baseline.
  • Tolerance: +/- 5% deviation from seller's estimate triggers a warning.
  • Scout must verify the seal integrity (if applicable).
User Experience (UX)
Two numeric inputs: Bag Count (Integer) and Total Net Weight (Decimal). Auto-calculates Average Bag Weight for anomaly detection (e.g., if one bag is too light).
User Actions
  • Enter Count & Weight.
  • Confirm: Transitions to parameter readings.
Technical Details
Validates `weight > 0` and `bag_count > 0`. Synchronizes with `lots` table updates.
Design Details
  • Large keypad-friendly inputs.
  • Summary chip showing calculated AVG weight.
Database Schema
UPDATE lots SET verified_weight = $1, verified_bag_count = $2, inspection_status = 'in_progress' WHERE id = $3;

Screen 2Parameter Readings
parameter_readings
Purpose
Direct capture of 6 quantitative quality metrics (Section 4.2).
Business Requirement
  • Metrics: 1. Moisture %, 2. Foreign Matter %, 3. Broken %, 4. Size (mm/suta), 5. Luster (Visual), 6. Hand-feel (Texture).
  • Moisture > 12% triggers a "Re-conditioning Required" hold.
User Experience (UX)
Grid of input tiles. Moisture and Foreign Matter require numeric entry. Luster and Hand-feel use 3-point rating chips (Excellent/Fair/Poor).
User Actions
  • Input Metrics: Must complete all 6 fields.
  • Guideline Link: Opens a popup with visual grade references.
Technical Details
Regex validation for % fields (0-100). Size grade mapped to DMI trading standard tiers.
Design Details
  • Input focus highlights in cyan.
  • Error state (Red) for out-of-bounds readings.
Database Schema
CREATE TABLE inspection_readings ( lot_id UUID REFERENCES lots(id), parameter_id TEXT, -- 'moisture', 'broken', etc. value_numeric NUMERIC, value_text TEXT );

Screen 3Defect Capture
defect_capture
Purpose
Log any qualitative deviations from the grade standard (Section 4.3).
Business Requirement
  • Coded Catalogue: Critical (Mold, Insect), Major (Khotta, Scorched), Minor (Undersized).
  • Any Critical = Automatic Rejection.
User Experience (UX)
Searchable defect list. Tapping a defect adds it to a "Found" list and requests an inline evidence photo.
User Actions
  • Select Defect: Categorized chooser.
  • Add Photo: Mandatory for each defect logged.
Technical Details
Defect severity lookup-table driven. State machine determines verdict eligibility in background.
Design Details
  • Red icons for Critical, Amber for Major.
  • Thumbnail previews of captured defect photos.
Database Schema
INSERT INTO lot_defects (lot_id, defect_code, severity, photo_path) VALUES (...);

Screen 5Reference Sample Log
reference_sample
Purpose
Document the physical drawing of a retention sample for 6-month quality arbitration (Section 4.5).
Business Requirement
  • 500g sample must be drawn and sealed.
  • Scout must log the Sample Bag ID.
  • Log the specific Shelf/Bin location for retrieval.
User Experience (UX)
Input for Sample Bag ID (QR scan enabled). Warehouse map picker for shelf location. Final photo of the sealed sample bag.
User Actions
  • Scan/Enter Bag ID.
  • Select Location.
  • Capture Sample Photo.
Technical Details
Barcode/QR scanning integration. Shelf location mapped to warehouse layout IDs.
Design Details
  • QR icon for scanning.
  • Location dropdown with breadcrumb path.
Database Schema

Screen 6Disposition Decision
disposition
Purpose
Final verdict engine determining the lot's commercial eligibility (Section 4.6).
Business Requirement
  • Verdict Logic: Any Critical -> Reject. Moisture > 12% -> Hold for Dry. Major within tolerance -> Grade B/C. Perfect -> Grade A.
  • Requires seller consent signature for grade downgrades.
User Experience (UX)
Verdict Banner (Grade A / B / C / Rejected). Summary of reason (e.g., "Downgraded due to Khotta %"). Inline signature pad for seller.
User Actions
  • Acknowledge Decision.
  • Collect Signature.
Technical Details
Calculates final grade client-side (for speed) and server-side (for truth). Verdict is immutable once signed.
Design Details
  • Gold/Silver/Bronze colors for Grades A/B/C.
  • Clean signature box with Clear/Save buttons.
Database Schema
UPDATE lots SET final_grade = $1, disposition_verdict = $2, seller_consent_signature = $3 WHERE id = $4;
