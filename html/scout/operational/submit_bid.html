Workflow Module 6

Submit Bid On Behalf

6-step wizard for submitting bids on buyer requirements on behalf of farmers/poppers. Filters by capability match, sets price and ready-by date, picks source pond/warehouse, and submits with Admin approval gate.

Bid Submission Steps
6 Steps
Step 1Choose User
Purpose
Select farmer or popper to submit bid on behalf of.
Data
Searchable list of active farmers/poppers scout has registered or is assigned to. Pending-approval users excluded.
Step 2Open Bid Invitations
Purpose
List requirements user is eligible to bid on.
Eligibility Filters
  • Capability match (user's farm type / warehouse location / capacity)
  • Grade tags (user's registered grade tags vs requirement tags)
Card Layout
Show requirement details: buyer, product spec, qty needed, delivery window, bidding deadline. Soft highlight if near deadline.
Step 3Choose Invitation
Purpose
Single-tap selection from available invitations.
Data
Requirement details populated for preview (buyer, product, qty, spec, delivery window).
Step 4Price Offer & Ready-By Date
Purpose
Capture seller's bid price and production ready-by date.
Data Inputs
  • Bid price (must be â‰¤ buyer's ceiling price)
  • Ready-by date (must be within buyer's delivery window)
Validation
Soft warnings on near-limit values (e.g., "Price is near ceiling" or "Ready date is near window end").
Step 5Lot Reference (Fulfillment Source)
Purpose
Select which pond/warehouse user will fulfill from.
Display
Cards of user's active sources (ponds for farmers, warehouses for poppers). Show area, capacity, recent activity.
Requirement
Must select exactly one source for bid fulfillment.
Step 6Confirm & Submit + Confirmation
What's Shown
  • Requirement summary (buyer, product, qty)
  • Bid price
  • Ready-by date
  • Source pond/warehouse
Atomic Submit
Creates bids record (status: pending_approval). Records queued to sync_queue. Notifies Admin.
Confirmation
Success state confirming bid submitted. Shows Admin approval required status. Single CTA back to Today Dashboard.
Admin Gate
Bid in pending state until Admin approves. User cannot transact until approval.

Bid Eligibility & Approval Flow

Eligibility Filters
Capability match (location, capacity, farm type) + grade tag match. Eligible invitations only shown to eligible users.
Price Ceiling Enforcement
Bid price must be â‰¤ buyer's maximum. Soft warning if near limit; hard block if exceeding.
Admin Approval Gate
Bid created as pending_approval. User cannot see bid as active until Admin approves. Prevents premature commitments.
Offline-First
All data captured locally; sync via sync_queue. Invitations list cached locally.
