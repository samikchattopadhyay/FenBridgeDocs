Backend Architecture & Implementation Guide

For Developers — Complete Specifications for Backend Implementation

tRPC Procedures (8 Screens)

Lot Verification involves progressive state accumulation inside the mobile wizard, finishing with an atomic database write transaction.

Precondition Guard:
• Entry requires warehouses.fitness_status = 'fit'. If unfit or unchecked, the workflow is hard-blocked.

Key Procedures

  • initLotVerification — Fetches active cached lot specifications, locked grade configurations, and target warehouse fitness states.
  • submitLotVerification — Protected procedure. Executes atomic multi-table write transaction. Inserts one inspections header, bulk inserts readings, defects, photos, and updates the child task states.

Database Schemas (Drizzle ORM DDL)

inspections (Header Table)

CREATE TABLE inspections ( id UUID PRIMARY KEY DEFAULT gen_random_uuid(), lot_id UUID NOT NULL REFERENCES lots(id), warehouse_id UUID NOT NULL REFERENCES warehouses(id), popper_id UUID NOT NULL REFERENCES poppers(id), scout_id UUID NOT NULL REFERENCES scouts(id), task_id UUID REFERENCES scout_tasks(id), related_po_id UUID REFERENCES purchase_orders(id), prior_inspection_id UUID REFERENCES inspections(id), -- for re-inspections declared_quantity_kg NUMERIC(10,2) NOT NULL, verified_quantity_kg NUMERIC(10,2) NOT NULL, verified_bag_count INT NOT NULL, quantity_variance_pct NUMERIC(6,2), claimed_grade TEXT, assessed_grade TEXT CHECK (assessed_grade IN ('A','B','C')), outcome TEXT NOT NULL CHECK (outcome IN ('accepted','downgrade','hold_admin','rejected')), outcome_locked BOOLEAN NOT NULL DEFAULT false, reject_reason_code TEXT, reject_recondition TEXT CHECK (reject_recondition IN ('reconditionable','terminal')), sample_ref TEXT, bags_sampled_from INT, retain_until DATE, overall_note TEXT, spec_version INT NOT NULL, checked_at TIMESTAMPTZ NOT NULL, created_at TIMESTAMPTZ NOT NULL DEFAULT now(), synced_at TIMESTAMPTZ ); CREATE INDEX idx_insp_lot ON inspections (lot_id, checked_at DESC);

inspection_parameter_readings (Quality Metrics)

CREATE TABLE inspection_parameter_readings ( id UUID PRIMARY KEY DEFAULT gen_random_uuid(), inspection_id UUID NOT NULL REFERENCES inspections(id) ON DELETE CASCADE, parameter_code TEXT NOT NULL, -- 'moisture'|'foreign_matter'|'broken'|'size_suta'|'colour_luster' value_numeric NUMERIC(7,2), value_coded TEXT, unit TEXT, band TEXT CHECK (band IN ('A','B','C','out')), spec_version INT NOT NULL );

Strict Input Validations & Error Enforcements

  • Hard Limits: Moisture locks in 0-25% bounds. Coded defect items map strictly against active catalogue lists.
  • Soft Warnings: Variance check triggers warning banner if abs(variance) > 5%, prompting scout verification before continuation.

Offline Caching & Queue Synchronization

  • Offline-First: Mobile client serializes full transaction packages into SQLite sync_queue if connectivity drops.
  • Server Sync Reconciliations: Reconciles local temporary sample IDs (e.g., SMP-temp) to final database formats (SMP-FY-XXXX) and fires PDF grade certificate creations asynchronously.

Security & Media Protection Protocols

  • RLS Access Policies: Scouts are constrained to operate only on lots housed within their geographical assignments.
  • Storage Protection: Camera photos compress client-side and upload directly to secure, authenticated Supabase buckets.
Lot Verification Steps Directory
8 Steps
1 Confirm Lot & Quantity
Purpose
Identifies the incoming lot and records the physical quantity verified by the scout against popper declarations.
Business Requirement

Verifies physical bag counts and net weight (kg).

Activates amber warning notice if net variance exceeds ±5%.

User Experience (UX)

Top panel shows lot reference metadata and linked PO.

Displays declared values.

Renders numeric entry fields for actual bag count and verified weight.

Inline chips display live calculated variance metrics.

Technical Details
Locks target spec_version in context, fetching the active specification tolerances before navigating downstream.
2 Parameter Readings
Purpose
Logs objective physical measurements for primary quality factors defined in the grade specification.
Business Requirement
Evaluates Moisture %, Foreign Matter %, Broken/Defective %, Size Grade (Suta), and Coded Colour Luster (Bright white / Acceptable / Dull-discoloured).
User Experience (UX)

Cards for each parameter displaying limits for A/B/C bands.

Inputting values instantly renders pass/near-limit/fail status colors.

A sticky footer reflects the provisional calculated grade in real-time.

Technical Details

Zod schemas block impossible ranges (e.g.

moisture > 25%).

Compiles readings into local context objects.

3 Defect Capture
Purpose
Checks the lot for critical, major, and minor physical defects from the standard quality framework catalogue.
Business Requirement

Locks severity tiers automatically based on defect code.

Critical defects (e.g.

mold, pest infestation, rodent contact) force immediate rejection verdicts.

User Experience (UX)

Default toggle reads "No defects found".

Toggling off unlocks bottom sheets displaying coded options categorized by severity.

Renders active summary badges showing the worst defect currently logged.

Technical Details

Failing to resolve defects while toggling off blocks navigation.

Critical/major defects dynamically append mandatory photo capture flags for Step 4.

4 Evidence Photos
Purpose
Generates robust photographic records supporting the quality grade and defending against downstream disputes.
Business Requirement

Mandatory slots: wide lot shot, close-up seed view, and moisture meter readout.

Dynamically includes mandatory slots for logged critical/major defects.

User Experience (UX)

Structured grid display showing dashed slots representing pending captures.

Active previews replace slots on snap.

Retake icons allow fast recaptures.

Continue button disabled until all mandatory slots are resolved.

Technical Details

Compresses files to ≤ 1.5MB.

Stamps GPS metadata.

Uploads to Supabase storage buckets.

5 Reference Sample
Purpose
Logs the physical dispute-reference sample collected and locked on-site by the scout.
Business Requirement

Nudges multi-bag draws.

Captures a mandatory picture of the locked sample bag.

Enforces 6-month retention policies.

User Experience (UX)

Generates distinct, copyable sample reference ID.

Prompts a numeric input asking for the bag count sampled.

Echoes read-only quality metric bands.

Technical Details

Assigns temporary sample_ref values, resolving final unique identifiers server-side.

Auto-calculates retain_until date (checked_at + 6 months).

6 Disposition
Purpose
Applies standard decision rules to calculate and verify the final lot verdict before database submission.
Business Requirement

Auto-evaluates disposition logic.

Accept (worst parameter band is A), Downgrade (assessed grade < claimed grade), Hold (majors out-of-spec), or Reject (critical defects present, locks outcome).

Commits atomic multi-table writes.

User Experience (UX)

Renders large outcome banners (Red warning for critical rejections, Amber warning for downgrades).

Displays detailed checklists echoing previous assessments.

Prompts mandatory confirmation checks.

Primary action reads "Submit Inspection".

Technical Details

Invokes atomic database transaction.

Mutates lots state column values.

Enqueues updates to the synchronization pipelines.

7 Grade Certificate Preview
Purpose
Visualizes the authoritative, auto-generated Grade Certificate that will be shared with the popper and prospective buyers.
Business Requirement

Ensures a valid preview can be generated reflecting the assessed grade, verified bags, quality metrics, and inline evidence photos.

(Skipped on Reject/Hold outcomes).

User Experience (UX)

Displays an interactive, read-only PDF view container.

Renders certificate identifiers.

Single active cyan button reads "Finish".

Technical Details
Renders an on-device local HTML mockup representing the server-side PDF generator template to ensure full offline operations.
8 Submission Confirmation
Purpose
Confirms completion and maps shortcuts for following task pipelines.
Business Requirement

Offers a direct "Apply seal now" shortcut button if the lot is accepted (and downgrade consent is not required/pending).

Otherwise routes back to the dashboard.

User Experience (UX)

Full-screen confirmation.

Shows sync statuses ("Synced" or "Offline Queued").

Banners display alerts indicating if downgrade notifications or reconditioning alerts were sent.

Cyan CTA maps onwards.

Technical Details

Clears all temporary draft context storage variables.

Marks local SQLite task items resolved.
