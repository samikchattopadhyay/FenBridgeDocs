tRPC Procedures (5 Screens)

Rejection Handling manages state forks between salvageable lots and absolute closures.

Triage Trigger Strategy:
• Written automatically by the Lot Verification wizard at Disposition submit when outcome = 'rejected'.
• If reject_recondition = 'terminal', sets lot status to rejected_terminal and inserts records into lot_closure_log.

Key Procedures

  • scout.rejection.getReport — Protected. Fetches failed parameter metrics, captured defect photos, and notes for a rejected lot.
  • scout.rejection.acknowledge — Protected. Invoked by poppers to record receipt of rejection notices. Mutates lots.popper_acknowledged_at.

React Component Decomposition & Reusability

This workflow breaks into reusable React components following atomic design principles. Components are shared across Scout module's 20 workflows.

✅ Design Philosophy:
• Single Responsibility: Each component does ONE thing
• Prop-based composition: Components accept data + actions as props
• No local state for business logic: Context + hooks for state
• Reuse across workflows: 80% of components shared across 20 workflows
• Variant management: Props define UI variants, not separate components

Database Schemas (Drizzle ORM DDL)

lot_closure_log (Permanent Closing Ledger)

CREATE TABLE lot_closure_log ( id UUID PRIMARY KEY DEFAULT gen_random_uuid(), lot_id UUID NOT NULL REFERENCES lots(id), closure_type TEXT NOT NULL CHECK (closure_type IN ('terminal_rejection','storage_recheck_terminal')), source_record_id UUID NOT NULL, -- inspections.id or storage_rechecks.id source_record_type TEXT NOT NULL CHECK (source_record_type IN ('inspection','storage_recheck')), reason_code TEXT NOT NULL, -- defect_code or storage failure code closed_at TIMESTAMPTZ NOT NULL, closed_by UUID REFERENCES users(id), -- scout id; null if system created_at TIMESTAMPTZ NOT NULL DEFAULT now() ); CREATE INDEX idx_lcl_lot ON lot_closure_log (lot_id);

lots (Columns added for acknowledgment tracking)

ALTER TABLE lots ADD COLUMN popper_acknowledged_at TIMESTAMPTZ, ADD COLUMN popper_acknowledged_by UUID REFERENCES users(id);

Row-Level Security (RLS)

-- Scouts can only insert closure records for lots they processed CREATE POLICY scout_closure_log_write ON lot_closure_log FOR INSERT WITH CHECK ( closed_by_scout_id = current_user_id() ); -- Terminal closure records are immutable CREATE POLICY closure_log_no_update ON lot_closure_log FOR UPDATE USING (false);

Strict Input Validations & Error Enforcements

  • Defect Catalogue Controls: Critical defects from the reference catalogue (infestation, rancidity, rodent contamination) are flagged non-salvageable, blocking reconditioning notice paths and forcing terminal closures.

Offline Caching & Queue Synchronization

  • Immediate Listing Halts: Synchronization of any rejected outcome automatically suspends corresponding market listings and sends instant push warning notices to popper accounts.

Security & Media Protection Protocols

  • Permanence: Terminal closure is irreversible. Closed lots cannot re-enter verification pipelines. Unique batch IDs are isolated to guard against re-submitting rejected stocks.
Rejection & Closure Views Directory
5 Screens
1 Rejection Report (Scout Detail View)
Purpose
Displays detailed quality failure metrics and defect records compiled during the inspection.
Business Requirement

Clearly illustrates the root reasons behind the failing grade.

Maps failing parameters (e.g. moisture 18%) against specs.

User Experience (UX)

Header shows red failure banner.

Displays parameters checklist indicating failed bands.

Renders grid containing defect evidence photos.

Prompts optional notes and coordinates.

Technical Details
Pulls data from active inspections records, mapping child defects directly.
2 Reconditioning Notice (Popper View)
Purpose

Guides poppers on fixing salvageable defects (e.g. wet makhana needing drying).

Business Requirement

Issued for reconditionable defects only.

Details specific guidelines (e.g., dry to moisture ≤ 12%).

Mutates lot state to pending_reinspection on popper acknowledgment.

User Experience (UX)

Renders yellow advisory banners.

Displays step-by-step drying/cleaning guidelines.

Active checkbox reads "I acknowledge rejection & will recondition lot".

Primary button triggers acknowledgment submission.

Technical Details
Logs acknowledgment timestamps into parent lots.
3 Terminal Notice (Popper View)
Purpose
Official notification to poppers stating that the lot has been permanently rejected and barred.
Business Requirement

Issued for critical unconditionable defects (infestation, rodent contact).

Sets status to rejected_terminal, barring the lot from any future listings.

User Experience (UX)

Full-screen danger warning.

Displays red warning badges indicating permanent bans.

Outlines specific safety hazard policies.

Popper is required to tap acknowledgment triggers to clear notification panels.

Technical Details
Coordinates and logs acknowledgment details, archiving records.
4 Chronological Quality History (Supporting View)
Purpose
Renders complete inspection chronology of a makhana lot in a single scroll timeline.
Business Requirement

Assures complete traceability history.

Lists first verification failures, re-conditioning dates, and reinspection results.

Walks the lineage indices.

User Experience (UX)

Vertical chronological flow lines connecting events.

Each node is a compact card displaying outcome status pills, inspection dates, and scout IDs.

Tapping cards opens corresponding report details.

Technical Details
Queries inspections joined on lot_id, ordering entries chronologically.
5 Terminal Closure Log (Admin View)
Purpose
Permanent record details of lot closing events available for administrative review.
Business Requirement
Preserves comprehensive close logs containing close type, source inspection records, reason code mappings, and timestamps.
User Experience (UX)

Detailed ledger row display.

Shows system sync status.

Displays secure warning logs details.

Search fields enable quick lookups.

Technical Details

Read-only view loading from lot_closure_log tables.

Marks lots closed permanently.
