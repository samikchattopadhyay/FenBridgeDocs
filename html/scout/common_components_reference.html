🔘 Buttons & CTAs

Primary interactive elements for user actions across all workflows.

Primary Button
Solid background, emerald green, white text. Used for main actions (Submit, Save, Confirm).
Classes: .btn-primary
Background: var(--accent-primary) #10B981
Padding: 12px 20px (small), 16px 28px (large)
Border-Radius: 8px
Used in Workflows:
Lot Verification Farmer Registration Popper Registration Pond Inspection
First Use: Farmer Registration submission step
Secondary Button
Ghost style with emerald green border and text. Used for secondary actions (Cancel, Back, Skip).
Classes: .btn-secondary
Border: 1px solid var(--accent-primary)
Background: transparent
Danger/Destructive Button
Used for irreversible actions like rejection or deactivation. Red/error color.
Classes: .btn-danger
Background: var(--error) #ffb4ab
Requires Confirmation: Always use with confirmation dialog
Used in: Rejection Handling, Deactivate Pond/Warehouse
Icon Button
Circular 44x44px minimum for tap targets. No text, icon only.
Min Size: 44x44px (iOS/Android accessibility)
Icon Size: 24px stroke, 2px weight

📝 Input Fields & Forms

Text, select, date, and file input components used across registration and data-capture workflows.

Text Input
Standard text field with label, optional error state, and helper text.
Background: #181818
Border: 1px solid #2D2D2D (default), transitions to var(--accent-primary) on focus
Label Position: Always above field
Padding: 12px 14px (minimum)
States:
default focus error disabled filled
Select Dropdown
Native or custom select for categorical choices. Always shows selected value clearly.
Used For: Status, Type, Category, Grade, Quality selection
Styling: Match text input styling
Examples: Grade selection (Lot Verification), Status (Farmer Management), Category (Rejection Handling)
Checkbox & Radio
For binary choices (checkbox) or single selection from options (radio).
Size: 18x18px minimum (touch target)
Label Alignment: Right of control
Spacing: 12px minimum between adjacent controls
Date/Time Picker
Native date input or custom picker. Always shows selected date in readable format.
Format: DD-MMM-YYYY (e.g., "15-May-2026")
Required Workflows: Farmer Registration, Popper Registration, Pond Inspection dates
Number Input
For numeric data: quantities, measurements, GPS coordinates, weights.
Decimals: 2 decimal places for measurements (weight, area)
Units: Always show unit label (kg, hectares, m², etc.)
Min/Max: Validate range based on context
Form Section Header
Groups related form fields. Provides visual and semantic grouping.
Style: Uppercase label, 12px font, text-dim color
Spacing: 20px margin-top between sections
Example Headers: "Personal Details", "Address Information", "Bank Account", "Pond Details"
Error Message
Field-level error text shown below input on validation failure.
Color: var(--error) #ffb4ab
Font Size: 12px
Position: Directly below input field
Example: "This field is required", "Invalid phone number", "Distance exceeds 50km"

📋 Cards & Containers

Primary vessels for content grouping and visual hierarchy.

Content Card
Standard container for form sections, data displays, workflow steps.
Background: var(--bg-card) #201f1f
Border: 1px solid var(--border-color)
Padding: 20px 24px (standard)
Border-Radius: 12px
Data Card (Lot/Farmer/Warehouse)
Displays key information for a single entity with optional actions. Clickable/expandable.
Layout: Header (ID/Name), body (key details), footer (status/actions)
Interaction: Tap to view details, swipe for actions
Used in: Lot Directory, Warehouse List, User Directory, Sync Queue
Summary Card
Displays aggregated or summary information. Read-only.
Use Case: Grade certificate, lot summary before submission, farmer summary
Background: Slightly darker (var(--bg-active)) to indicate read-only state
Highlighted/Alert Card
Draws attention to important information, warnings, or status changes.
Types: Success (green), Warning (yellow), Error (red), Info (blue)
Left Border: 3px colored border
Background: Subtle color overlay (10% opacity)
Examples: Rejection notice, lot approved status, sync conflict indicator
Side-by-Side Comparison Card
Shows before/after or current/target values for inspection comparisons.
Layout: Two columns with centered divider
Example: Prior seal condition vs. current condition in Storage Re-Check

🏷️ Status & Badges

Visual indicators for state, approval status, and workflow progress.

Status Badge
Inline label showing current status. Small, non-interactive.
Format: Rounded background with text
Padding: 4px 8px
Font Size: 11px, weight 600
Status Types (Color Coded):
Draft (Gray) Pending (Yellow) Approved (Green) Rejected (Red) Verified (Green) Sealed (Blue)
Progress Badge
Shows completion percentage or step progress within a workflow.
Format: "3 of 7 steps" or "45% complete"
Use Case: Multi-step registration (farmer, popper)
Verification Badge
Indicates data has been verified or is pending verification.
States: Verified ✓, Pending ⏳, Failed ✗
Color: Green (verified), Yellow (pending), Red (failed)
Used in: Farmer Registration (OTP, Penny-drop), Photo verification steps
Category Chip
Labels or tags for categorization (defect types, lot categories, etc.)
Format: Small pill with text
Padding: 4px 10px
Use Case: Defect types (Discoloration, Damage, Size Variation), Quality grades

📊 Data Tables & Lists

For displaying structured data across multiple rows with consistent formatting.

List Item (Minimal)
Single row with title and optional subtitle/metadata. Clickable for navigation.
Layout: Title (bold), subtitle (muted), with optional right indicator (status badge or arrow)
Height: Minimum 48px touch target
Divider: 1px solid #2D2D2D between items
Used in: User Directory, Warehouse List, Pond List
Data Row (With Details)
Multi-column row displaying structured entity data.
Columns: Max 4-5 columns on mobile, more on desktop
Alignment: Left-align text, right-align numbers
Spacing: 12px horizontal padding per column
Table Header
Column labels for data tables. Sticky on scroll.
Background: var(--bg-active) with 1px bottom border
Font: 12px, weight 600, uppercase
Sticky: Position sticky when list scrolls
Empty State
Message shown when no data available. May include action button.
Content: Icon (48px), title, description, optional CTA
Example: "No lots to verify yet" with "Create Lot" button

📷 Media Components

Photo capture, upload, and display components critical to Scout verification workflow.

Photo Capture Card
Interaction point for capturing photos via device camera or uploading from gallery.
Layout: Dashed border box with camera icon and "Tap to capture" text
Size: 120x120px minimum (touch target)
State After Capture: Shows thumbnail of captured image
Workflows: Lot Verification (evidence photos), Seal Application, Storage Re-Check, Pond Inspection, Rejection Handling
Photo Gallery/Grid
Displays multiple captured or uploaded photos in a grid layout.
Layout: 2-3 columns on mobile
Interaction: Tap to view full-screen, swipe to delete
Count Badge: Shows number of photos captured (e.g., "3/5 required")
Photo Viewer (Full-Screen)
Modal or full-screen view of a single photo with metadata.
Controls: Close button, prev/next arrows, zoom controls
Metadata: Timestamp, GPS location (if available), file size
Document Preview
Displays document images (ID, business cert, etc.) with ability to rotate/zoom.
Use Case: Farmer Registration (Aadhar, bank statements), Popper Registration (business docs)
Features: Rotation, zoom, download
Photo Upload Queue
Shows pending photos waiting to sync in offline workflow context.
Indicator: "3 photos pending sync" with spinner icon
Location: Typically in Sync Queue or bottom of screen

🗺️ Navigation & Steppers

Components for guiding users through multi-step workflows and screens.

Stepper/Progress Indicator
Horizontal step indicator showing current position in multi-step workflow.
Layout: Numbered circles (1, 2, 3...) connected by lines
Active State: Highlighted circle with glow effect
Completed State: Checkmark in circle
Used in: Farmer Registration (13 steps), Popper Registration (16 steps), Lot Verification (7 steps)
Tab Navigation
Multiple content sections in single screen. Switch between tabs to view different data.
Layout: Horizontal tabs with underline indicator
Active Indicator: Emerald green underline
Use Case: Lot details (inspection, seal, re-check history)
Breadcrumb Navigation
Shows current page location in hierarchy. Allows quick navigation up the tree.
Format: "Scout > Lots > Lot #123"
Separator: ">" or "/" icon
Last Item: Non-clickable (current page)
Bottom Navigation (Mobile)
Fixed navigation bar at bottom of screen for primary workflows.
Items: 3-5 main sections (Dashboard, Workflows, Users, Sync, Profile)
Active State: Icon highlighted in emerald green
Badge: Notification count badge (e.g., "3" for pending approvals)
Contextual Back Button
Navigates back to previous screen. Always visible, typically top-left or top-right.
Style: Icon button (chevron left or back arrow)
Size: 44x44px minimum

💬 Dialogs & Modals

Modal interactions for confirmation, alerts, and focused task completion.

Confirmation Dialog
Asks for user confirmation before destructive action (reject lot, deactivate user, etc.)
Content: Icon, title, description, action buttons
Buttons: Primary action (destructive red) + Cancel
Accessibility: Focus starts on Cancel button (safer default)
Triggers: Reject Lot, Deactivate Pond/Warehouse, Delete User, Mark Ready
Success/Status Modal
Shows result of completed action (submission successful, data saved, etc.)
Content: Success icon (checkmark), title, optional reference ID, CTA
Auto-dismiss: 2-3 seconds OR requires explicit close
Used After: Lot submission, Farmer registration complete, Seal application submitted
Error Modal
Displays error message to user with option to retry or dismiss.
Content: Error icon, error message, suggestion for fix
Buttons: "Retry", "Dismiss", or "Go Back"
Bottom Sheet / Drawer
Modal that slides up from bottom. Used for options menus or secondary forms.
Content: Title, list of actions/options, close button (X or outside tap)
Backdrop: Semi-transparent dark overlay dismisses sheet on tap
Toast/Snackbar
Transient notification at bottom of screen. Auto-dismisses after 3-4 seconds.
Use Case: Quick feedback (copied to clipboard, data saved, connection restored)
Types: Success (green), Info (blue), Warning (yellow), Error (red)

⚙️ State Indicators & Feedback

Components for showing loading states, network status, offline mode, and validation feedback.

Loading Spinner
Circular animated spinner indicating data is loading.
Size: 24px (inline), 48px (full-screen)
Color: var(--accent-primary) emerald green
When to Show: API calls > 300ms, file uploads, image processing
Skeleton Loader
Placeholder shapes showing where content will appear. Better UX than spinner.
Pattern: Gray animated boxes matching content layout
Use Case: Lot list loading, user details, warehouse info
Network Status Bar
Always-visible indicator showing online/offline status.
Position: Top of screen (below app bar)
States: "Online" (green), "Offline" (red), "Syncing" (yellow)
Required In: All Scout screens (essential for offline-first app)
Critical for: Sync Queue, Mobile App
Validation Feedback
Real-time feedback on form field validity.
Valid State: Green checkmark next to input
Error State: Red error icon + error message below input
Timing: On blur or real-time (do not validate until first submit)
Sync Progress Indicator
Shows progress of background sync operation (photos, forms, data).
Format: Progress bar with "3 of 10 items syncing"
Location: Sync Queue screen or as bottom bar
Actions: Pause, Resume, View Details
Conflict Indicator
Alerts user to data conflicts during offline-to-online sync.
Visual: Red badge or warning box
Action: Opens conflict resolution UI
Required for: Offline-first architecture
