Workflow Module 5

Warehouse Fitness Check

One-time and periodic warehouse suitability verification. Scout assesses 6 coded criteria (3 critical) to determine if warehouse can hold BM lots. Fail on any critical criterion blocks warehouse from holding inventory; notifies Admin.

Fitness Assessment Steps
5 Screens
Screen 1Confirm Warehouse & Arrival
Purpose
Confirms scout is at correct warehouse; captures arrival GPS as evidence.
Data Shown
  • Warehouse identity card (name, popper, address)
  • Prior fitness check history (if any)
  • GPS capture block with mismatch warning
Validation
Warns if GPS mismatch >200m from registered warehouse location.
Screen 2Fitness Checklist
Six Criteria (Pass / Fail Toggle)
  • ✓ Dry / Cool / Shaded (no direct sun, no heat) — Non-critical
  • ⚠ Stock off-floor (wooden pallets/platforms) — CRITICAL
  • ⚠ No pest/rodent signs — CRITICAL
  • ⚠ Chemical & odour separation (no pesticides nearby) — CRITICAL
  • ✓ Roof & walls sound (no leaks, damp) — Non-critical
  • ✓ Adequate ventilation (not humid/closed) — Non-critical
User Experience
  • Critical criteria marked with ⚠ badge
  • Each fail row expands coded-reason chooser
  • Live summary chip shows Pass/Fail count
Requirement
All 6 must be assessed. Every fail must have coded reason selected from dropdown.
Screen 3Evidence Photos
Purpose
Mandatory photographic evidence supporting checklist assessment.
Required Photos
  • Interior wide-angle (showing full warehouse layout)
  • Storage/stacking detail (showing bag placement, pallets)
  • One auto-generated photo per failed criterion (e.g., pest evidence)
  • Optional: extra context photos
Validation
Cannot proceed until all required slots filled. Photo compression ≤1.5MB each; GPS-stamped.
Screen 4Verdict & Submit
Verdict Logic
  • If any critical criterion failed: verdict LOCKED as FAIL
  • If only non-critical criteria failed: scout decides PASS or FAIL
  • If all pass: verdict PASS (locked, computed)
What's Shown
  • Final verdict (PASS / FAIL)
  • Full checklist recap with reasons for each fail
  • Photo thumbnail strip
  • Optional overall notes (max 500 chars)
  • Confirm checkbox before submit
Critical Warning
If FAIL: Scout sees message: "Warehouse FAILS inspection. This warehouse cannot hold BM lots until issues are resolved. Admin notified."
Screen 5Submission Confirmation
Success State
Shows verdict (PASS / FAIL), warehouse name, sync status, single CTA: "Back to Warehouses."
Backend Action
  • Creates fitness_checks record with verdict (pass / fail)
  • If FAIL: warehouse status → fitness_failed; notifies Admin immediately
  • If PASS: warehouse status → fitness_passed; allows lot sealing
  • Records queued to sync_queue
Impact on Lots
FAIL blocks new lot sealing at this warehouse. Existing sealed lots remain valid. PASS enables sealing.

Critical Requirements & Enforcement

Critical Criteria (3)
Off-floor stacking, No pests, Chemical separation. Any fail = FAIL verdict, warehouse blocked.
Photo Evidence Mandatory
2 fixed photos + 1 per failed criterion + optional extras. Cannot submit without all required filled.
Admin Notification
FAIL triggers immediate Admin alert. Scout sees clear messaging about consequence (warehouse blocked).
Offline-First
100% offline capable. Photos compressed during capture. Data synced via sync_queue.
